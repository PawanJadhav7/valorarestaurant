# GitHub Workflow for Valora AI

## Standard Flow
1. Create or update an issue
2. Add labels, priority, milestone, and project fields
3. Create a branch from the issue
4. Make the change locally
5. Commit and push the branch
6. Open a pull request
7. Link the PR to the issue
8. Wait for CI and review
9. Merge to main
10. Confirm the issue closes

## Branch Naming
- feature/...
- fix/...
- docs/...
- chore/...

## Pull Request Rule
All changes to main must go through a pull request with review.
